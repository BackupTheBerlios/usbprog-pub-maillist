Firstly I would like to apologise that this post is not in German, I
have not come across an English speaking mailing list for the USBProg.

I have tried to follow some posts regarding the ATTiny2313 and get the
vague impression that other people may be experiencing similar issues.

I have found that with the default fuses for the 2313 (hfuse 0xDF, lfuse
0x64) I am unable to communicate with the 2313 using the USBProg.
However using a ponyser device there appear to be no problems.

Using the ponyser device I then set the fuses to their operational
values of hfuse 0xDF, lfuse 0xEF. I can now communicate with the 2313
using the USBProg device.

I have also noticed that the USBProg in conjunction with avrdude 5.4
under Linux 2.6.22 (libusb 2:0.1.12) only seems to work on the third
attempt each time.


prompt> sudo avrdude -p t2313 -P usb -c avrispmkII -v

 avrdude: usbdev_open(): did not find any USB device "usb"

prompt> sudo avrdude -p t2313 -P usb -c avrispmkII -v

 avrdude: usbdev_open(): Found AVRISP mkII Klon, serno: 0000A00128255
(pauses for quite some time)

prompt> sudo avrdude -p t2313 -P usb -c avrispmkII -v

Works straight away

When the USBProg does work it is incredibly fast!

I would like to offer any help I can in tracking down these issues.
