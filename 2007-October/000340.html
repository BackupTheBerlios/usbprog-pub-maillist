>> >> >
>> >> >As XSVF instructions can become quite long I think it's best to send
>> >> > one instruction at a time via usb. This way usbprog can also send back
>> >> > information to the player software running on the computer wether the
>> >> > current XSVF instruction was successful or not.

>> >> Yes you can use the simpleport firmware as draft.

>> >> >Is it correct that a USB packet has a maximum payload size of 1023

>> >> further hacks.
