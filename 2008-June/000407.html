Hi,

as you may have already noticed, Windows installation support for
USBprog is poor. Not because the program has bugs (well, I'm sure it
has, but that's unrelated to Windows), not because the GUI doesn't work
on Windows, only because the libusb installation is crappy.

So, if you have different Windows versions (VMs may work, but I had no
luck with VirtualBox and QEmu on Linux host and Windows guest with
USBprog hardware) and are able to read a big NSIS documentation, then
you might be the developer we look for.

You don't have to touch the source files of USBprog, you don't have to
be a C/C++ specialist, you just should have time and Windows experience.
Currently I know two bugs:

- Vista does not work
- you manually have to specify the path of sys and inf file when Windows
  detects the new hardware

If you're able to fix that bugs and send me patches to the installation
NSIS script, than please drop me a note at <bernhard.walle at gmx.de>.

Thanks for your help!
